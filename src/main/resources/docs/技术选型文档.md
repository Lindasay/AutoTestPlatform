# 企业级自动化测试平台（MySQL版）技术选型文档
## 一、选型原则
1. 无学习成本：优先选择已掌握的技术栈，14天内完成MVP开发
2. 企业级落地：贴合大厂测开岗位实际技术栈，避免小众技术
3. 轻量易扩展：核心功能封装，后续可快速扩展新能力
4. 面试友好：技术选型可讲透设计思路，突出对标岗位竞争力

## 二、核心技术栈
### 2.1 后端核心
| 技术         | 版本       | 选型理由                                                                 |
|--------------|------------|--------------------------------------------------------------------------|
| Spring Boot  | 2.7.18     | 稳定版，生态完善，企业级后端开发标配，无学习成本                         |
| MyBatis-Plus | 3.5.3.1    | 简化MySQL操作，支持自动填充、分页、逻辑删除，比原生MyBatis效率更高       |
| MySQL        | 8.0        | 替代H2内存库，贴合企业生产环境，支持复杂查询、索引优化，面试可讲性能调优 |

### 2.2 自动化核心
| 技术         | 版本       | 选型理由                                                                 |
|--------------|------------|--------------------------------------------------------------------------|
| RestAssured  | 5.3.0      | 接口自动化主流工具，支持JSON/XML解析，封装性强，比HttpClient更易用       |
| TestNG       | 7.8.0      | 测试用例管理框架，支持数据驱动、分组执行，适配自动化用例编排             |
| Playwright   | 1.41.0     | UI自动化工具，大厂趋势，支持多浏览器，比Selenium更轻量，失败自动截图     |

### 2.3 工程化&工具链
| 技术         | 版本       | 选型理由                                                                 |
|--------------|------------|--------------------------------------------------------------------------|
| Spring Task  | 内置       | 轻量级任务调度，支持多线程异步执行，无需额外部署中间件（如XXL-Job）       |
| Allure       | 2.24.0     | 测试报告工具，支持定制化模板，可视化效果好，是面试亮点                   |
| Maven        | 3.8+       | 依赖管理+项目构建，企业级标准，便于后续CI/CD整合                         |
| Git          | 最新版     | 版本管理，规范提交记录，体现工程化思维                                   |
| Lombok       | 最新版     | 简化实体类代码，减少模板代码编写                                         |
| Hutool       | 5.8.22     | 通用工具类库，封装日期、加密、Excel解析等能力，避免重复造轮子             |

### 2.4 前端
| 技术         | 选型理由                                                                 |
|--------------|--------------------------------------------------------------------------|
| Thymeleaf    | 与Spring Boot无缝整合，无需前后端分离，快速开发简易页面，节省时间         |
| 原生HTML/CSS/JS | 仅实现基础交互，不追求美观，聚焦后端核心能力打磨                         |

### 2.5 CI/CD
| 技术         | 选型理由                                                                 |
|--------------|--------------------------------------------------------------------------|
| Jenkins      | 最新版     | 开源CI/CD工具，支持Pipeline脚本，可实现“拉取代码→构建→测试→生成报告”全流程 |

## 三、技术栈对标岗位
1. 大厂资深测开/专家：MySQL性能优化、Allure定制化、CI/CD流程是核心考察点
2. 中小厂测试经理：平台架构设计、任务调度、自动化用例封装是核心竞争力
