# 企业级自动化测试平台（MySQL版）四层架构设计文档
### 一、架构设计目的（对标大厂标准）
1. **解耦性**
    - 各层严格单向依赖（Controller -> Service -> Mapper -> Common）,禁止夸层调用（如Controller直接访问Mapper）
    - 通过接口抽象隔离变化（如Service层定义接口，实现类独立变化）
2. **可维护性**
    - 分层职责清晰：
        * **表现层：**仅处理HTTP请求/响应，不包含业务逻辑
        * **业务层：**封装核心用例管理、任务调度等逻辑
        * **数据层：**专注MySQL数据库操作，使用使用MyBatis-Plus简化CRUD
        * **基础层：**提供通用工具（如分页、异常处理），避免代码重复。
    - 新人可通过架构图快速定位代码位置
3. **可扩展性**
    - 新增模块（如性能测试）只需在对应层扩展
        * 表现层新增`PerformanceController`。
        * 业务层实现`PerformanceService`。
        * 数据层添加`PerformanceMapper`和表结构。
    - 支持多数据源（如未来接入Redis缓存用例）。
4. **面试友好**
    - 可清晰阐述：
        * 为什么选择四层架构（对比传统MVC的臃肿问题）。
        * 如何通过MyBatis-Plus实现数据层高效开发。
        * 工程化规范（如CheckStyle、多环境配置）如何保障可维护性。

#### 2.2 分层详解与技术选型
| 层级 | 职责 | 核心类/技术 | 面试可讲点 |
| --- | --- | --- | --- |
| **表现层（Controller）** | 1. 接收前端HTTP请求（如JSON/表单）<br/>2. 参数校验（如@Valid）<br/>3. 调用Service处理业务<br/>4. 返回统一格式响应（如Result<T>） | - `ProjectController.java`   - `RestController`   - `@RequestBody`   - `Result<T>`（通用返回封装） | - 为何不直接返回MySQL实体？   （答：防止暴露敏感字段，如`status`的数据库值0/1需转换为"禁用/启用"）   - 如何防止XSS攻击？   （答：通过`StringUtils.escapeHtml()`工具类过滤前端输入） |
| **业务层（Service）** | 1. 实现核心业务逻辑（如用例批量执行、任务调度算法）<br/>2. 事务管理（如@Transactional）<br/>3. 调用Mapper操作数据库<br/>4. 组合多表查询结果（如用例+项目信息） | - `ProjectService.java`   - `@Service`   - `@Transactional`   - `BaseService`（通用CRUD抽象） | - 为何需要`BaseService`？   （答：避免每个Service重复写`save()`/`delete()`等方法，提升代码复用）   - 如何保证接口自动化用例的幂等性？   （答：通过数据库唯一索引+重试机制） |
| **数据层（Mapper）** | 1. 定义MySQL操作接口（如查询项目列表）   2. 使用MyBatis-Plus简化SQL（如`LambdaQueryWrapper`）   3. 复杂查询通过XML或注解实现 | - `ProjectMapper.java`   - `BaseMapper<Project>`   - `@Mapper`   - MyBatis-Plus（分页插件、乐观锁） | - 为何选择MyBatis-Plus而非JPA？   （答：更灵活的SQL控制，适合复杂查询如按标签筛选用例）   - 如何优化慢查询？   （答：为`project_id`和`tags`字段添加联合索引） |
| **基础层（Common）** | 1. 提供通用工具（如日期处理、分页工具）   2. 全局异常处理（统一返回错误码）   3. 常量定义（如用例状态0-未启用，1-启用）   4. 代码规范约束（如Lombok简化Setter/Getter） | - `Result<T>.java`   - `GlobalExceptionHandler.java`   - `DateUtils.java`   - `Constants.java`   - Lombok（`@Data`注解） | - 为何封装`Result<T>`而非直接返回实体？   （答：统一响应格式，包含状态码、消息、数据，便于前端处理）   - 如何处理全局异常？   （答：拦截`NullPointerException`等，返回友好提示如"用例不存在"） |
